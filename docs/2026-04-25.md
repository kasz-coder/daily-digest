## OFFICIAL BLOGS

**Anthropic Engineering: An update on recent Claude Code quality reports**

Anthropic published a candid postmortem on the quality issues users reported over the past month. Three separate bugs were identified: (1) a reasoning effort change that defaulted to medium instead of high, hurting coding quality — reverted April 7; (2) a caching bug that continuously dropped Claude's prior thinking for the rest of a session after 1 hour of inactivity, making it forgetful and repetitive — fixed April 10; and (3) a system prompt instruction capping text to 25 words between tool calls that degraded intelligence — reverted April 20. The aggregate effect of three staggered changes looked like broad degradation even though the API was unaffected. As a remedy, Anthropic is resetting usage limits for all subscribers and adding tighter controls on system prompt changes going forward — broader eval suites, per-model ablations, and gradual rollouts for any change that trades off intelligence.
[https://www.anthropic.com/engineering/april-23-postmortem](https://www.anthropic.com/engineering/april-23-postmortem)

**Claude Blog: Built-in memory for Claude Managed Agents**

Claude Managed Agents now ships with memory in public beta. Agents can learn across sessions using a filesystem-based memory layer — developers can export, manage, and audit memories via the API. Early adopters include Netflix agents carrying context across sessions, Rakuten's task agents cutting first-pass errors by 97%, and Wisedocs speeding up document verification by 30%. Memory files are portable and rollback-capable, giving enterprise teams full control.
