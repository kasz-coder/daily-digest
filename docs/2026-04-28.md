**Box CEO Aaron Levie** argues that the fear of AI-driven job loss is overblown. Most people use AI for their own jobs and see all the nuances and complexities up close — the data access issues, the context needed to get models to work, the output review, the business process integration. Then they see AI do something in a different domain and assume it can automate that entire function immediately. "We tend to dramatically underestimate the work that goes into making the AI work just as effectively in those jobs."

Levie also noting two subtle factors contributing to the current sense of overwork with agents: (1) The leverage on incremental effort has gone up substantially — ICs now get a similar sense to people managers about maximizing team time, making prioritization a key skill. (2) AI makes it so much easier to start incremental projects that end up taking much longer than realized. He regularly starts a project at 9PM thinking it'll be quick and finds himself at midnight completing it.

**Anthropic Engineering: Claude Code auto mode — a safer way to skip permissions**

Auto mode is a new mode that delegates approvals to model-based classifiers — a middle ground between manual review and no guardrails. It uses two layers: a prompt-injection probe at the input layer scanning tool outputs before they enter the agent's context, and a transcript classifier at the output layer evaluating each action before execution.

The classifier runs in two stages: a fast single-token filter (tuned to err toward blocking), followed by chain-of-thought reasoning only if the first filter flags the action. On real internal traffic (n=10,000), Stage 1 has 8.5% false positive rate. With Stage 2 added, FPR drops to 0.4% at the cost of 17% false negative rate on real overeager actions.

Key insight from their incident log: agents deleting remote git branches from misinterpreted instructions, uploading engineers' GitHub auth tokens to internal clusters, attempting migrations against production databases. "Each of these was the result of the model being overeager, taking initiative in a way the user didn't intend."
