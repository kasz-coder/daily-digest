**Anthropic Engineering: Scaling Managed Agents — Decoupling the brain from the hands**

The takeaway: Anthropic's new Managed Agents service virtualizes agent components (session, harness, sandbox) into general interfaces, following the same pattern operating systems used to virtualize hardware — allowing implementations to be swapped without disturbing the interfaces.

Anthropic realized their early approach of coupling everything into a single container created "pets" rather than "cattle" — if a container failed, the session was lost. They solved this by decoupling the "brain" (Claude and its harness) from the "hands" (sandboxes that perform actions) and the "session" (the event log). The harness now calls containers as tools: execute(name, input) → string. If a container dies, the harness catches the failure and can spin up a new one. They also moved credentials outside the sandbox to prevent prompt injection attacks from reaching sensitive tokens.

The performance gains were significant: p50 time-to-first-token dropped ~60% and p95 dropped over 90%, because inference could start immediately without waiting for container provisioning. The architecture also allows one brain to connect to many "hands" — execution environments that Claude reasons about and routes work to.

Key quote: "The challenge we faced is an old one: how to design a system for 'programs as yet unthought of.' Operating systems have lasted decades by virtualizing the hardware into abstractions general enough for programs that didn't exist yet."
